# Plan 18: Check-in Templates

**Status: ✅ Complete** · **Priority: P0** · **Effort: Low** · **Impact: High**

## Problem

Users often repeat similar check-ins (e.g. "PR reviews", "Feature work", "Bug fixes"). Preset templates would speed up the Build Summary flow.

## Tasks

1. [x] Define 3–5 default templates (e.g. "PR reviews", "Feature work", "Meetings", "Bug fixes")
2. [x] Add template selector or quick-insert buttons above the check-ins textarea
3. [ ] Store templates in localStorage so users can add/edit (optional)
4. [x] One-click insert template text into form

## Success Criteria

- User can insert a preset check-in with one click
- Templates are visible and easy to discover

---

## Resolution

- 5 default templates: PR reviews, Feature work, Bug fixes, Meetings, Documentation
- Quick-insert buttons above check-ins textarea; one-click appends to field
