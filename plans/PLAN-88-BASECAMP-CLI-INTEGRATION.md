# Plan: Basecamp CLI Integration

**Status: ✅ Complete** · **Priority: P1**

## Problem

Daily check-in text and weekly summaries are generated by the app but manually copy-pasted into Basecamp. There is also no way to review past Basecamp check-in answers from within the app.

## Changes

### 1. Post to Basecamp on Capture / Build Summary

- `lib/basecamp-post.ts` – wraps `basecamp` CLI to post daily check-in answers and weekly summaries
- `api/snapshot` and `api/summary` routes accept a `postToBasecamp` flag
- UI toggle in `TodaySection` and `FullSummaryForm` (conditionally rendered when Basecamp env vars are set)
- Environment variables: `BASECAMP_PROJECT_ID`, `BASECAMP_CHECKIN_QUESTION_ID`, `BASECAMP_WEEKLY_QUESTION_ID`

### 2. Check-ins Explorer

- `lib/basecamp-fetch.ts` – fetches check-in answers via CLI, filters to current user, sanitizes Basecamp HTML
- HTML sanitizer transforms `<bc-attachment>` mentions into inline `@Name` spans and image attachments into proxied inline images
- `app/routes/api.basecamp-image.ts` – server-side image proxy using `basecamp files download --out -` (Basecamp storage URLs require session auth, not Bearer tokens)
- `app/routes/checkins.tsx` – date-grouped timeline page with lightbox for inline images
- Nav link added to `app/root.tsx`
